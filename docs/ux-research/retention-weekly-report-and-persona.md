# Retention, Weekly Report, and Persona Voice (v1)

목표: 사용자가 SoloBoss AI를 계속 사용하게 만드는 심리적 가치와 반복 사용 루프를 설계한다.

## 1) Retention Strategy

## Core Psychological Value

- `나는 관리받고 있다`
- `내 매출이 보인다`
- `지금 무엇을 해야 할지 안다`

## Retention Loop

1. 월요일 아침: 지난주 성과를 숫자로 요약한다.
2. 즉시 행동: 이번 주 우선 연락 대상 1~3명을 제시한다.
3. 실행 피드백: 사용자의 실제 실행 결과를 다시 다음 리포트에 반영한다.

## Product Positioning

- 톤: `코치형`
- 리포트 목표: `균형형(매출 + 즉시 행동)`
- 수치 노출 정책: `혼합형(숫자 공개 + 추정치 라벨)`

## 2) Weekly Report Design

## Delivery Timing

- 발송 시점: 매주 월요일 오전 `08:30 KST`
- 실패 재시도: 15분 간격 2회
- 3회 실패 시: 웹 콘솔 `주간 리포트함` 적재

## Message Structure (30초 내 스캔)

1. 헤드라인: 지난주 성과 한 줄
2. 숫자 블록: 상담 수, 예상 매출, 전주 대비
3. 액션 블록: 이번 주 우선 연락 대상 1~3명
4. 코치 멘트: 실행 지시 1문장
5. CTA: `지금 팔로업 시작`

## AlimTalk Example Copy (Coach Voice)

```text
[이번 주 영업 코칭 리포트]

지난주 성과: 상담 5건, 예상 매출 500만원(+18%)
좋아요. 이번 주는 전환만 챙기면 됩니다.

우선 행동(오늘):
1) 박OO - 3일 무응답, 계약확률 80%(추정)
2) 김OO - 견적 확인 대기, 계약확률 72%
3) 이OO - 결제 일정 문의 필요, 계약확률 65%(추정)

코치 한마디:
오늘 3명만 처리하면 주간 목표 달성 가능성이 높습니다.
지금 1번부터 연락하세요.
```

## Metric/Scoring Logic

- `expected_revenue = Σ(건별 금액 × 계약확률)`
- `win_probability` 입력:
  - 모델 점수
  - 최근 응답 패턴
  - 영업 단계(견적/협의/결제 전)
- 신뢰도 낮은 값은 `(추정)` 라벨 표시
- 전주 대비는 절대값 + 퍼센트 동시 표시

## Contact Priority Rule

- 우선순위 점수:
  - `priority_score = win_probability * potential_amount * urgency_decay`
- 최근 24시간 내 이미 연락한 고객은 감점
- 본문에는 상위 3명만 노출

## 3) Persona Voice and Tone for Errors

## Persona Definition

- 역할: `성과를 챙겨주는 영업 코치`
- 성격: 친근하지만 실행 지향적
- 금지: 기계적 오류 코드 중심 문구, 과도한 사과, 책임 회피

## Message Formula

1. 상황 인정(짧게)
2. 원인 설명(사용자 탓 금지)
3. 지금 할 행동 1개 제시
4. 확신/동기부여 한 문장

## Do / Don't

- Do
  - 짧은 명령형 CTA 사용
  - 실패를 다음 행동으로 전환
  - 이모지 0~1개 사용
- Don't
  - "`에러 코드 123`"만 전달
  - "`지원팀 문의`"로 끝내기
  - 장문 사과문

## Error Message Standards

- 텍스트만 전송:
  - "좋아요, 내용은 받았어요. 그런데 이번엔 캡처/음성이 필요합니다. 상담 캡처 1장만 보내주시면 바로 등록해드릴게요."
- 흐린 이미지:
  - "앗, 제 눈이 침침해서 글자가 흐려요 📸 조금 더 밝은 곳에서 한 장만 다시 찍어주세요. 이번엔 정확히 정리하겠습니다."
- 노출 불량:
  - "이미지는 도착했어요. 다만 밝기가 강해서 텍스트가 가려졌습니다. 밝기만 살짝 조정해 다시 보내주세요."
- 일반 사진:
  - "좋은 사진인데 상담 텍스트가 보이지 않아요. 카톡 대화가 보이는 캡처로 보내주시면 바로 CRM에 넣겠습니다."
- 다중 전송/순서 꼬임:
  - "자료 확인했습니다. 순서가 섞여 해석이 불안정해요. 핵심 캡처 1장부터 보내주시면 순서대로 정리하겠습니다."

## Duplicate Input Tone

- 기본 문구:
  - "같은 상담으로 판단해 기존 기록에 합쳤습니다. 필요하면 이번 건만 분리 저장할 수 있어요."
- 원칙:
  - 결과 중심, 비난 금지, Undo 옵션 즉시 제공

## 4) KPI for Retention Validation (MVP)

- 주간 리포트 열람률
- 리포트 수신 후 24시간 내 팔로업 실행률
- 우선순위 고객 3명 처리율
- 주간 예상매출 대비 실제 계약 전환률
- 4주 유지율(연속 4주 리포트 수신/활동)
